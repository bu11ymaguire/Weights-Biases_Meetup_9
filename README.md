## 주요 내용

### 1. 생성형 AI와 컴퓨팅 패러다임의 변화

 생성형 AI가 단순한 기술 트렌드를 넘어 컴퓨팅 패러다임 자체를 변화시키고 있다는 관점이 제시되었다.

**기존 소프트웨어 (결정론적 시스템)**
- 명확한 규칙(Rule) 기반
- 사용자가 UI를 학습하고 특정 기능을 실행
- 입력에 대한 출력이 예측 가능

**생성형 AI 기반 소프트웨어 (확률론적 시스템)**
- 사용자의 의도(Intent)를 해석하여 결과 생성
- AI가 도구에서 협업자로 역할 변화
- 개발자에게 요구되는 역량이 코딩 기술을 넘어 확장됨

교육과정상 알고리즘과 명확한 로직 설계를 다루고 있는데, 확률적 시스템 설계가 중요한 과제로 다뤄지고 있다는 점이 인상적이었다.

---

### 2. LLM의 환각(Hallucination) 문제와 해결 접근

LLM이 생성하는 정보의 신뢰성 문제, 특히 환각 현상에 대한 논의가 있었다. 현업에서는 이를 해결하기 위해 다양한 기술적 접근을 시도하고 있다.

**주요 해결 방법**
- RAG(Retrieval-Augmented Generation): 검색을 통한 사실 기반 답변 생성
- Knowledge Graph 활용: 구조화된 지식 참조
- EntiGraph 등 최신 연구 동향

**언어별 환각 빈도 차이**
- 영어: 학습 데이터 풍부, 환각 빈도 상대적으로 낮음
- 한국어: 데이터 부족으로 환각 빈도 높음
- 한국어 특화 모델 및 데이터셋 구축의 필요성

수업에서는 주로 머신러닝, 딥러닝의 알고리즘 위주로 공부를 하는데 실제는 데이터의 양이 가장 중요하다는 것을 다시 한번(1학기 ML 프로젝트를 하면서 처음 느꼈었다.) 느끼게 되었다.   

---

### 3. LLM 모델별 편향(Bias) 분석

각 LLM 모델이 동일한 데이터를 받아도 서로 다른 응답 성향을 보인다는 분석 결과가 흥미로웠다. 주식 투자 의견(매수/매도)을 모델별로 비교한 히트맵이 제시되었는데,

- Llama4: 매수 의견이 압도적으로 많음 (낙관적 성향)
- Claude: 보수적이고 신중한 답변 (중립적 성향)

같은 질문에 대해 모델마다 고유한 편향이 존재한다는 것은, 실무에서 모델을 선택할 때 단순히 벤치마크 성능만이 아니라 모델의 특성과 편향을 함께 고려해야 함을 의미한다.
